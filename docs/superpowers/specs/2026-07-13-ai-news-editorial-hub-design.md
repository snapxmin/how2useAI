# AI 资讯编辑策展型门户设计

## 背景

现有 `/news` 页面以双列资讯卡片为主，能够展示来源、分类和摘要，但缺少明确的视觉焦点、内容层级和站内深度内容。改版参考 36Kr 首页的信息组织思路，但不复制其品牌视觉或具体组件。

目标是把 AI 资讯页升级为编辑精选的信息门户：用户既能快速掌握最新动态，也能沉浸阅读大厂关键技术、产品和行业趋势解读。

## 已确认的产品方向

- 页面定位：编辑精选的信息门户。
- 阅读去向：站内原创解读为主，快讯跳转权威外部来源。
- 首版范围：完成新布局，并补充一批可展示的站内解读样例。
- 布局方案：头条、快讯和深度内容并存的编辑策展型布局。

## 目标与成功标准

### 目标

1. 首屏同时表达“今日最重要内容”和“最新动态”。
2. 为大厂技术路线、产品变化和能力对比提供稳定的站内内容入口。
3. 建立从浏览资讯到阅读解读、订阅周报的连续路径。
4. 保持桌面端内容丰富度，同时保证移动端阅读节奏和可用性。

### 成功标准

- 用户在首屏能区分头条、快讯和深度解读。
- 站内内容与外部链接具有明确且一致的视觉标识。
- 页面在无头条、无专题或无快讯时仍能正常展示。
- 桌面端无明显空洞区域，移动端无横向页面溢出。
- 分类 URL 可分享、可被搜索引擎索引。

## 信息架构

页面按以下顺序组织：

1. **分类导航**：全部、大模型、产品动态、技术解读、行业趋势等。
2. **今日焦点**：一条站内深度解读，承担首屏视觉焦点。
3. **24 小时快讯**：展示最新外部资讯和发布时间。
4. **大厂技术与产品拆解**：三条站内解读，覆盖技术路线、产品观察和能力对比。
5. **最新资讯**：按时间倒序展示完整资讯流。
6. **本周专题**：串联同一主题下的资讯、解读和实战指南。
7. **订阅入口**：将一次访问转化为持续阅读。

桌面端首屏使用不对称双栏：今日焦点占主栏，快讯占侧栏。后续区域采用“主内容 + 专题侧栏”的编辑布局。移动端改为单列，焦点内容优先，快讯使用横向滑动卡片，其余内容按优先级依次排列。

## 视觉与交互

### 视觉语言

- 延续现有品牌蓝，以深海军蓝作为主背景、亮蓝作为交互强调色。
- 使用浅灰页面背景和白色内容卡片，提高长时间浏览的舒适度。
- 橙色仅用于专题等少量辅助区域，不与主品牌色竞争。
- 减少大面积渐变和装饰，让标题、摘要和编辑判断成为视觉中心。
- 卡片使用轻描边、有限圆角和克制阴影，形成科技媒体而非工具目录的观感。

### 交互原则

- 分类条保持在页面顶部附近，支持快速切换主题。
- 卡片悬停只使用轻微位移、描边和标题变色。
- 不使用自动轮播，避免打断阅读。
- 站内解读显示阅读时长，外部内容显示来源和外链标识。
- 移动端隐藏低价值摘要，优先保留标题、时间和来源。
- 所有交互元素具有可见的键盘焦点状态。

## 内容模型

### 快讯

快讯继续存储在 `content/news/news.json`，并沿用现有字段：

- `id`
- `title`
- `summary`
- `source`
- `url`
- `date`
- `category`
- `tags`
- `featured`

快讯负责时效性，点击后跳转权威来源。

### 深度解读

深度解读使用 MDX 存储。Frontmatter 至少包含：

- `title`
- `description`
- `date`
- `updated`（可选）
- `category`
- `insightType`：技术路线、产品观察、能力对比或行业趋势
- `companies`
- `products`
- `tags`
- `readingTime`
- `featured`
- `heroTheme`

文章正文需要回答“发生了什么、为什么重要、技术或产品发生了哪些实质变化、对用户有什么影响”。

### 首页精选配置

首页精选配置存储在 `content/news/featured.json`，负责指定：

- `heroInsightSlug`：今日焦点的解读 slug。
- `insightSlugs`：大厂拆解条目的解读 slug 数组，最多三条。
- `weeklyTopic`：包含标题、描述以及关联资讯、解读和指南 slug 的专题对象。

如果配置缺失或引用无效内容，查询层使用确定性的最新内容进行回退。

首版补充 3–4 篇深度解读样例，不接入实时抓取、数据库或 CMS。

## 技术架构

### 页面和路由

- `app/news/page.tsx`：服务端资讯门户页。
- `app/news/insights/[slug]/page.tsx`：站内深度解读详情页。

详情页复用现有指南的 MDX 解析、目录和相关文章模式，避免建立第二套渲染体系。

### 数据访问

- 扩展 `lib/news.ts`，提供快讯排序、分类筛选、精选和回退查询。
- 新增 `lib/insights.ts`，负责读取、校验和排序 MDX 解读。
- 分类由 `searchParams` 驱动，服务端输出筛选结果。

数据流为：

`本地 JSON/MDX → 服务端读取与校验 → 排序和精选回退 → 页面区块组件`

### 组件边界

- `NewsCategoryNav`：分类导航和当前分类状态。
- `NewsHero`：今日焦点。
- `BreakingNews`：24 小时快讯。
- `InsightGrid`：大厂技术与产品拆解。
- `NewsFeed`：按时间展示快讯。
- `WeeklyTopic`：本周专题。

每个组件只接收已经整理好的展示数据，不在组件内部读取文件或决定全局精选逻辑。

## 异常处理与降级

- 没有配置头条时，选择最新的 featured 深度解读；仍为空时选择最新解读。
- 没有专题时隐藏专题区块，并让资讯流占满可用宽度。
- 没有快讯时展示简短空状态，不影响深度内容。
- 首页精选配置引用不存在的 slug 时忽略该引用并执行回退。
- 外部 URL 不合法时不渲染可点击链接，并在内容校验中报告错误。
- 单篇 MDX 解析失败时，构建校验应给出文件路径和字段信息。

## 测试与验收

### 自动化测试

- 快讯按日期倒序排列。
- featured 内容和最新内容回退顺序正确。
- 分类筛选结果正确，未知分类回退到全部。
- 首页配置中的无效引用不会导致页面失败。
- 空快讯、空专题和空解读状态可正常渲染。
- 站内解读使用内部路由，快讯使用带安全属性的外部链接。
- MDX 必填字段、日期、重复 slug 和非法 URL 能被检测。

### 人工验收

- 桌面端首屏信息主次明确。
- 移动端无页面级横向溢出，快讯滑动区可正常操作。
- 键盘可以访问所有链接和分类入口，焦点可见。
- 站内与站外跳转在点击前可以被识别。
- 运行测试、Lint 和生产构建。

## 非目标

- 不实现实时新闻抓取、自动摘要或推荐算法。
- 不建设后台 CMS、用户账号、收藏或评论系统。
- 不复制 36Kr 的品牌资产、文案、配色或具体组件。
- 不在首版加入无限滚动或自动轮播。
