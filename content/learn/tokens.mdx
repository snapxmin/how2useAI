## 日常问题

为什么一段中文、英文和代码占用的“上下文长度”不同？因为模型不会直接读取文字，而会先用分词器把内容切成 Token。

<ConceptCard title="Token">
Token 是模型处理文本的基本单位。它可能是一个汉字、词的一部分、标点或代码符号，具体切法取决于模型的分词器。
</ConceptCard>

## 背后的真实机制

文本先被分词器转换为 Token 编号，再变成向量进入神经网络。模型最终输出的也是下一个 Token 的概率分布，然后再还原成人能阅读的文字。

Token 数影响三个现实问题：

1. 输入与输出能否放进上下文窗口。
2. API 使用成本。
3. 模型处理长材料时的速度与注意力分配。

## 这会怎样影响工具使用

不要只看文件页数。重复内容、冗长日志和无关附件都会消耗 Token，让真正重要的信息更难被关注。先清理材料，再给 AI。

<Misconception title="一个字就是一个 Token">
不同模型的分词器不同，同一段文字的 Token 数也可能不同。估算可以帮助规划，但精确数量应以具体工具显示为准。
</Misconception>

## 小实验

<PracticeTip>
把同一句话分别写成中文、英文和带变量名的代码，让支持 Token 统计的工具比较数量。观察标点与生僻词如何被切分。
</PracticeTip>

## 三句话总结

1. 模型先把文本切成 Token 才能计算。
2. Token 数决定可容纳内容、速度和成本。
3. 精简无关上下文比单纯“塞更多资料”更有效。
