## 日常问题

普通聊天一次问答就结束，编码 Agent 为什么会搜索文件、修改代码、运行测试、再继续修复？因为它运行在一个可重复的反馈循环中。

<ConceptCard title="Agent 循环">
感知当前状态 → 制定或更新计划 → 选择并执行动作 → 读取工具结果 → 判断目标是否完成。未完成就继续，达到停止条件或需要人工决定时退出。
</ConceptCard>

## 背后的真实机制

Agent 通常由模型、指令、工具、状态和控制器组成。控制器负责把工具结果送回模型、限制循环次数、处理失败和决定何时请求人工确认。

规划不一定一次完成。可靠 Agent 会根据真实结果调整下一步，而不是盲目执行最初计划。任务越长，越需要明确检查点和可观察的中间产物。

## 这会怎样影响工具使用

适合 Agent 的任务通常目标清楚、步骤可验证、工具可访问、错误可恢复。目标模糊、后果不可逆或需要价值判断的任务，应保持人在回路中。

<Misconception title="Agent 就是更聪明的大模型">
Agent 的差异主要来自系统结构：它能保存状态、调用工具并循环。底层模型仍会误判，循环甚至可能把一次错误放大。
</Misconception>

## 小实验

<PracticeTip>
给 Agent 一个可验证的小任务，要求先列计划，每完成一步报告证据，遇到删除、发送或付款动作必须暂停确认。观察它如何根据工具反馈调整。
</PracticeTip>

## 三句话总结

1. Agent 是围绕目标运行的模型、工具、状态与控制循环。
2. 反馈让它可以修正计划，而不只生成一次答案。
3. 明确停止条件、检查点和人工审批比“完全自主”更可靠。
