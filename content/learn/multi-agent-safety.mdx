## 日常问题

让多个 Agent 分工是否一定比一个更好？只有当任务可以清晰拆分、结果可以整合并且并行收益大于协调成本时，多 Agent 才有价值。

## 背后的真实机制

多 Agent 系统会给不同角色分配上下文和工具，例如研究、执行、审查。它可以并行探索或互相检查，但也会引入通信丢失、重复劳动、错误共识和更高成本。

<Misconception title="让两个 AI 互相审核就等于事实正确">
如果它们使用相同错误来源或相似模型，可能自信地达成错误共识。独立证据和确定性验证比角色数量更重要。
</Misconception>

## 人机协作的安全边界

按动作风险设置自治等级：

1. **建议**：AI 给方案，人决定。
2. **草拟**：AI 生成待审内容，人批准后发布。
3. **受限执行**：AI 在沙盒或限定范围内操作。
4. **高风险操作**：必须双重确认、最小权限、日志和回滚。

隐私同样属于边界。不要把不必要的个人信息、商业机密和密钥交给模型；了解服务的数据保留与训练政策。

## 把原理变成五个使用习惯

<ConceptCard title="结课行动清单">
1. **正确选模型**：按准确性、速度、成本和隐私选择，而非只看“最强”。  
2. **拆分任务**：把模糊目标拆成可验证步骤。  
3. **补足上下文**：提供事实、边界、样例和验收标准。  
4. **允许调用工具**：需要实时信息或执行动作时连接合适工具，并限制权限。  
5. **人工验证关键结果**：错误后果越高，证据、审批和回滚越严格。
</ConceptCard>

## 小实验

<PracticeTip>
选一个真实任务，写下：目标、输入资料、可用工具、每步验收标准、哪些动作必须人工确认。然后再决定用普通聊天、单 Agent 还是多 Agent。
</PracticeTip>

## 三句话总结

1. 多 Agent 用协调成本换取并行与专业分工，并非默认更好。
2. 安全来自最小权限、独立证据、审批、日志与回滚。
3. 真正会用 AI，是把模型原理转化为稳定的工作习惯。
