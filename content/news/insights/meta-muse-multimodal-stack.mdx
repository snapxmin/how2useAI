## 发生了什么

Meta 连续发布 Muse Spark 1.1、Muse Image，并预览 Muse Video。与此同时，Meta Model API 开放预览，让开发者可以通过统一入口使用相关模型能力。

这些发布覆盖工具调用、代码、计算机操作、图像生成编辑以及原生音视频生成。

## 为什么重要

多模态产品的竞争正在从单一模型效果转向能力组合。真正的生产流程往往同时需要理解文本和素材、生成内容、编辑结果，并把能力接入应用。

模型 API 的出现使这些能力更容易被智能体调用，也让 Meta 的发布从几个独立模型变成一套可组合的工具栈。

## 关键技术或产品变化

Muse Spark 强调任务理解和工具调用，Muse Image 负责精确编辑与多参考图组合，Muse Video 则扩展到原生音频视频生成。

这种分工并不意味着用户必须直接操作多个模型。更可能的产品形态，是由上层智能体判断何时调用哪种能力，再统一管理输入和输出。

## 对使用者的影响

评估多模态工具时，不应只比较单张图片或单段视频的效果，还要检查编辑可控性、接口可用性、来源标记和工作流集成成本。

对开发团队而言，API 的稳定性、调用权限和输出追踪会直接决定这些模型能否进入正式生产环境。
