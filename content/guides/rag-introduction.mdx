## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是让 AI **基于你的私有数据**回答问题，而不是仅依赖训练时的公开知识。

### 一个类比

- **普通 ChatGPT**：像一个博览群书但不知道你公司情况的顾问
- **RAG 系统**：像一个熟悉你公司所有文档的专属助理

### 工作原理

```
用户提问 → 从知识库检索相关文档 → 将文档作为上下文传给 AI → AI 基于文档生成回答
```

三个核心步骤：
1. **文档入库**：将 PDF、Word、网页等文档切片并建立索引
2. **检索**：用户提问时，找到最相关的文档片段
3. **生成**：AI 基于检索到的内容生成精准回答

## 什么时候需要 RAG

| 场景 | 普通 ChatGPT | RAG |
|------|-------------|-----|
| 通用知识问答 | 够用 | 没必要 |
| 基于公司内部文档问答 | 回答错误或编造 | RAG 准确回答 |
| 客服机器人 | 只能说通用话术 | 基于产品文档精准回答 |
| 法律/医疗等严谨领域 | 风险高 | 有据可查，降低幻觉 |

## 实战：用 Dify 搭建知识库

[Dify](https://dify.ai) 是最适合初学者的 RAG 平台，开源且支持本地部署。

### 第一步：创建知识库

1. 登录 Dify → 知识库 → 创建
2. 上传文档（支持 PDF、Word、Markdown、网页）
3. 配置切片策略：
   - 切片大小：500-1000 字符
   - 重叠：50-100 字符（避免上下文断裂）

### 第二步：创建 AI 应用

1. 应用 → 创建空白应用 → 聊天助手
2. 关联刚创建的知识库
3. 配置 Prompt 模板：

```
你是 [公司名] 的内部知识助手。请基于检索到的文档回答用户问题。

规则：
1. 只基于提供的文档内容回答，不要编造
2. 如果文档中没有相关信息，明确说「根据现有文档，未找到相关信息」
3. 回答时引用文档来源
```

### 第三步：测试与优化

用以下问题测试系统：

- 文档中**明确有答案**的问题 → 检查回答是否准确
- 文档中**没有答案**的问题 → 检查是否正确拒绝回答
- **模糊表述**的问题 → 检查检索是否找到正确文档

### 优化方向

- **检索不准**：调整切片大小，或换用更好的 Embedding 模型
- **回答不完整**：增加检索返回的文档数量（Top K）
- **回答太慢**：减少 Top K，或使用更快的模型

## 进阶路线

掌握基础 RAG 后，可以探索：

- **混合检索**：关键词 + 语义检索结合，提升准确率
- **多轮对话**：让 RAG 系统记住上下文，支持追问
- **Agent + RAG**：让 AI 不仅能查文档，还能执行操作（如发邮件、创建工单）
