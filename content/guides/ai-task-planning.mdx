## 适合哪些任务

当你面对一个目标，但不知道从哪里开始时，可以先让 AI 帮你拆解。典型场景包括：

- 新项目启动前，拆分里程碑和任务
- 接到临时需求后，评估先做什么
- 制定周计划、月计划和复盘计划
- 把领导的一句话目标变成可执行动作

AI 擅长提供结构化初稿，但最终优先级仍需要结合真实资源和业务判断。

## 基础 Prompt：目标拆解

```
请帮我把以下目标拆解成可执行计划。

## 目标
[写清楚你要完成什么]

## 当前情况
- 已有资源：[人员、资料、工具]
- 时间限制：[截止日期]
- 约束条件：[预算、审批、依赖团队]

## 输出格式
1. 阶段划分：每个阶段的目标和交付物
2. Todo 清单：按优先级排序
3. 风险提醒：可能卡住的地方
4. 今天就能开始的 3 个动作

## 要求
- 每个任务要具体到可以直接执行
- 不要写“持续跟进”这类模糊动作
```

## 进阶用法一：判断优先级

当任务很多时，可以让 AI 按影响度和紧急度排序。

```
请帮我整理以下任务优先级。

## 任务列表
[粘贴任务]

## 判断标准
- 影响业务结果的程度
- 是否有明确截止时间
- 是否依赖其他人
- 是否会阻塞后续任务

## 输出要求
请分为：
1. 今天必须做
2. 本周必须推进
3. 可以延后
4. 需要先确认信息

每个任务后说明排序理由。
```

## 进阶用法二：提前发现风险

计划最容易失败的地方，是忽略依赖和边界。可以让 AI 专门从反面检查。

```
请站在项目经理视角，审查以下计划。

## 计划内容
[粘贴计划]

## 请重点检查
1. 哪些任务描述不够具体
2. 哪些任务缺少负责人或截止时间
3. 哪些地方存在跨部门依赖
4. 哪些风险可能导致延期
5. 哪 3 个问题应该马上确认
```

## 人工确认清单

AI 生成计划后，不要直接照做。建议再确认：

- 任务是否真的有负责人
- 截止时间是否现实
- 是否遗漏审批、数据、素材等依赖
- 是否把“大任务”拆到了 1-2 小时可推进的粒度

## 推荐工具

- **豆包**：适合快速生成日常工作计划和清单
- **Kimi**：适合复杂任务拆解和长文档上下文
- **ChatGPT**：适合任务拆解、优先级排序和风险分析
- **Notion AI**：适合把计划沉淀到项目文档

一个好的 AI 任务规划，不是让计划看起来更复杂，而是让下一步行动更清楚。
