## 为什么适合用 AI

职场中很多时间花在“读资料、找重点、整理结论”上。AI 很适合处理这类重复性阅读任务：

- 阅读长文档、政策、制度、合同初稿
- 总结群聊、邮件往来和客户反馈
- 整理行业资料和竞品信息
- 从多份材料中提取共性问题

AI 的价值不是替你做最终判断，而是先帮你把信息压缩成可读、可行动的版本。

## 三种常用整理方式

### 方式一：快速摘要

适合第一次阅读一份长材料时使用。

```
请阅读以下材料，并帮我做一份快速摘要。

## 输出格式
1. 一句话结论
2. 5 个核心要点
3. 对我最重要的 3 个影响
4. 需要继续确认的问题

## 要求
- 不要遗漏数字、日期和责任方
- 如果原文没有明确说明，请标注“原文未说明”

## 材料
[粘贴资料内容]
```

### 方式二：提取行动项

适合处理聊天记录、会议记录、邮件往来。

```
请从以下内容中提取可执行事项。

## 输出格式
- 事项：
- 负责人：
- 截止时间：
- 依赖条件：
- 风险提醒：

## 要求
- 只提取明确提到的事项
- 负责人或时间不明确时，用 [待确认] 标注
- 合并重复事项

[粘贴聊天记录/邮件记录]
```

### 方式三：对比多份资料

适合做调研、选型、竞品分析。

```
我会提供多份资料，请帮我对比整理。

## 对比维度
1. 核心观点
2. 适用场景
3. 优势
4. 风险或限制
5. 对我的建议

## 输出要求
- 先给结论，再给依据
- 用清晰的小标题组织
- 不确定的地方不要猜测

[粘贴多份资料]
```

## 使用注意事项

1. **重要资料要保留原文链接**：方便回查上下文，避免只看摘要误判。
2. **敏感信息先脱敏**：客户名、金额、合同条款等信息尽量替换成占位符。
3. **让 AI 标注不确定项**：比让它给“完美答案”更可靠。

## 推荐工具

- **Kimi**：适合长文档和多资料整理
- **豆包**：适合快速摘要日常文档和中文素材
- **DeepSeek**：适合技术文档和结构化信息提取
- **Claude**：适合复杂文档理解和结构化总结
- **Perplexity**：适合带来源的公开资料调研

用好这个流程，可以把“先读一遍再整理”的工作，变成“先让 AI 摘要，再人工判断重点”。
